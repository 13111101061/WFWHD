# 项目目录结构说明

面向接手同事的快速导读，入口采用 `apps/api` 布局，业务代码仍在 `src/`。

## 顶层
- `apps/api/`：运行入口与路由装配、静态资源
  - `index.js`：Express 入口，挂载所有 API
  - `routes/`：路由层（只做装配、调用 src 中的模块）
  - `public/`：静态页面（demo/admin 等）
- `src/`：业务与基础能力（待逐步下沉到 packages/*）
  - `core/`：认证、监控等核心中间件
  - `modules/`：业务模块（tts、snpan、admin、sms[Paused]）
  - `shared/`：config、middleware、utils、repositories
  - `storage/`：本地存储（cache、data、uploads）
- `packages/`：分层占位（core-auth、monitoring、tts、storage、shared、sms），尚未迁移代码
- `tests/`：测试主目录（integration/e2e/fixtures/debug +各类 *.js 用例）
- `examples/`：示例/调试脚本与 HTML
- `scripts/`：运维/工具脚本
- `sdk/`：SDK 示例（原“sdk文件/”）
- `docs/`：文档与重构建议
- `var/`：运行期产物（音频、缓存、上传、数据），已在 `.gitignore` 中忽略
- `.claude/`：本地工具配置
- `package.json`：项目依赖与脚本

## 当前约定
- 入口固定在 `apps/api/index.js`；路由层只引用 `src/` 下业务代码。
- 运行产物统一放 `var/`，不要回写到根目录。
- 后续如需分层，可将 `src/*` 逐步迁入 `packages/*`，但保持路由层薄、业务在包内。
