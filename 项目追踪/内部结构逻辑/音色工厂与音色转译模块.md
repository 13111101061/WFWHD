## VOICE_DATA_MODEL

### Voice object in VoiceRegistry (raw):
```json
{
  "id": "moss-tts-ashui",
  "provider": "moss",
  "service": "tts",
  "sourceId": "ashui",
  "displayName": "阿树",
  "gender": "female",
  "languages": ["中文"],
  "tags": ["松弛", "耐听", "治愈", "温暖"],
  "description": "冬天清晨的灰白天空...",
  "runtime": {
    "voiceId": "2001257729754140672",
    "voice": "2001257729754140672",
    "model": "moss-tts",
    "providerOptions": {
      "samplingParams": { "temperature": 1.7, "top_p": 0.8, "top_k": 25 }
    }
  },
  "ttsConfig": {
    "sourceId": "2001257729754140672",
    "model": "moss-tts",
    "samplingParams": { "temperature": 1.7, "top_p": 0.8, "top_k": 25 }
  }
}
```

### Field meaning:
| Field | Purpose | Example |
|-------|---------|---------|
| `id` | System voice ID (unique, used by frontend/API) | "moss-tts-ashui" |
| `sourceId` | Short name alias | "ashui" |
| `runtime.voiceId` | Provider real voice ID (what the external API needs) | "2001257729754140672" |
| `runtime.voice` | Alias for voiceId (same value) | "2001257729754140672" |
| `runtime.model` | Model identifier for this provider | "moss-tts" |
| `runtime.providerOptions` | Provider-specific runtime params | samplingParams |
| `ttsConfig` | Legacy config format (backward compat) | same as runtime |

## VOICE_ID_TRANSLATION_PIPELINE (v3.0)

### Input (from HTTP request):
```json
{
  "text": "你好测试",
  "service": "moss_tts",
  "voice": "moss-tts-ashui"
}
```
- `voice` can be: system ID ("moss-tts-ashui"), short name ("ashui"), or provider real ID ("2001257729754140672")

### Translation steps (VoiceResolver.resolve()):
```
1. Normalize request
   - Extract voiceId from: request.voiceId, request.voice, options.voiceId, options.voice
   - Result: voiceId = "moss-tts-ashui"

2. Resolve canonical service key
   - ProviderCatalog.resolveCanonicalKey("moss_tts") -> "moss_tts"

3. Lookup voice in voiceRegistry
   - voiceRegistry.get("moss-tts-ashui") -> raw voice object

4. Extract runtime data
   - voiceRuntime = rawVoice.runtime
     -> voiceId: "2001257729754140672"
     -> providerOptions.samplingParams: {...}
   - Fallback: if no runtime, build from ttsConfig via _buildRuntimeFromConfig()

5. Normalize runtime
   - Flatten providerOptions into top-level
   - Ensure voice/voiceId alias sync

6. Build final runtimeOptions (MERGE ORDER - CRITICAL):
   merged = {
     ...baseOptions (defaults from ttsDefaults),
     ...normalizedRuntime (from voiceRegistry runtime),
     ...userOptions (from request options)
   }

7. LOCK voice/voiceId from runtime (HIGHEST PRIORITY):
   if (normalizedRuntime.voiceId || normalizedRuntime.voice) {
     merged.voice = normalizedRuntime.voice || normalizedRuntime.voiceId;
     merged.voiceId = normalizedRuntime.voiceId || normalizedRuntime.voice;
   }
   -> This ensures user cannot override voice ID with wrong value

8. Result passed to provider adapter:
   runtimeOptions.voice = "2001257729754140672" (provider real ID)
   runtimeOptions.samplingParams = {...} (from voice config)
```

### Output (to provider API):
```json
{
  "model": "moss-tts",
  "text": "你好测试",
  "voice_id": "2001257729754140672",
  "sampling_params": { "max_new_tokens": 20000, "temperature": 1.7, "top_p": 0.8, "top_k": 25 },
  "meta_info": true
}
```

## VOICE CATALOG CONVERSION (VoiceCatalog.toCatalogVoice):
```
raw voice (from VoiceRegistry)
  │
  ├─► profile: { id, displayName, gender, languages, tags, description, ... }
  │     -> for display/listing
  │
  └─► runtime: { voiceId, voice, model, sampleRate, providerOptions, ... }
        -> for synthesis execution
```

## PROVIDER-SPECIFIC NOTES

### MOSS-TTS:
- Requires numeric voice_id (e.g. "2001257729754140672")
- Endpoint: https://studio.mosi.cn/api/v1/audio/speech
- Required params: model, text, voice_id
- Recommended: sampling_params.max_new_tokens = 20000 (not 512)
- Optional: expected_duration_sec, meta_info (defaults to true)

### Aliyun Qwen HTTP:
- Endpoint: https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation
- Request format: { model, input: { text, voice } }
- Voice name is string (e.g. "Cherry", "Ethan")
- Does NOT support speed/volume/pitch adjustment

### Aliyun CosyVoice (WebSocket):
- WebSocket connection to dashscope
- Voice name is string (e.g. "longxiaochun_v2")

## V3_CHANGES
- VoiceResolver now reads runtime directly from voiceRegistry (not VoiceCatalog.getRuntime())
- runtime.voiceId has HIGHEST priority, cannot be overridden by user params
- Added _buildRuntimeFromConfig() fallback for legacy voice data
- VoiceCatalog.getRuntime() was returning null due to conversion issue -> bypassed
