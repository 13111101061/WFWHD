## PRIMARY_CALL_GRAPH
0. Domain + ports define contracts (`src/modules/tts/domain/*`, `src/modules/tts/ports/*`).
1. POST `/api/tts/synthesize` -> `ttsRoutes.js` (merged, v3.0)
2. TtsHttpAdapter.synthesize(req, res) -> TtsSynthesisService.synthesize(request)
3. TtsSynthesisService:
   a. _validateRequest() -> SynthesisRequest.validate() + TtsValidationService
   b. _resolveServiceIdentifier() -> parse service string or lookup by systemId
   c. _checkRateLimit() -> per-service rate limiter (100 req/min)
   d. circuitBreaker.execute(() => _synthesizeWithRetry())
      - timeout: 60s (env: TTS_SYNTH_TIMEOUT_MS)
      - retry: 1 attempt with exponential backoff (120ms * attempt)
      - retryable errors: API_ERROR, PROVIDER_ERROR, TIMEOUT_ERROR, network errors
4. TtsProviderAdapter.synthesize(provider, serviceType, text, options)
5. VoiceResolver.resolve() - voice ID translation:
   a. User sends voice="ashui" (system voice ID)
   b. voiceRegistry.get("ashui") -> raw voice data
   c. Extract runtime.voiceId = "2001257729754140672" (provider real ID)
   d. runtime.voiceId has HIGHEST priority, NOT overridden by user params
   e. Merge: baseOptions <- runtimeOptions <- userOptions (voice/voiceId locked from runtime)
6. createProvider(adapterKey) -> specific adapter instance
7. adapter.synthesizeAndSave(text, runtimeOptions):
   a. _getCredentials() -> credentials.selectCredentials() (pool + health check)
   b. Call external provider API (HTTP/WebSocket)
   c. _reportSuccess() / _reportFailure() -> health tracking + circuit breaker
   d. audioStorageManager.saveAudioFile() -> save to disk
8. AudioResult.fromServiceResult() -> DTO normalization
9. TtsHttpAdapter returns standardized JSON response

## VOICE_ID_TRANSLATION_RULE
- Frontend sends: voice = system voice ID (e.g. "moss-tts-ashui" or "ashui")
- Provider needs: voice_id = provider real voice ID (e.g. "2001257729754140672")
- Translation pipeline:
  1. VoiceResolver.resolve() looks up voice in voiceRegistry by system ID
  2. Extracts runtime.voiceId (provider real ID) from voice data
  3. Merges into runtimeOptions with HIGHEST priority
  4. Provider adapter uses runtimeOptions.voice as voice_id
- Data source structure:
  ```json
  {
    "id": "moss-tts-ashui",
    "sourceId": "ashui",
    "runtime": {
      "voiceId": "2001257729754140672",
      "voice": "2001257729754140672",
      "model": "moss-tts",
      "providerOptions": { "samplingParams": {...} }
    },
    "ttsConfig": { "sourceId": "2001257729754140672", ... }
  }
  ```

## CREDENTIAL_POOL_FLOW
- config_source: YAML files in `credentials/sources/providers/*.yaml`
- .env_policy: NO provider API keys in .env; only service-level config
- init:
  - try YAML loader first (`credentials/sources/providers/*.yaml`)
  - fallback to `.env` default account (deprecated path)
- runtime:
  - `selectCredentials(provider, service, context)` -> best account by priority + health
  - execute provider call
  - `reportSuccess` / `reportFailure` -> health tracking + circuit breaker
  - circuit: 5 consecutive failures -> account disabled, retry after 60s
