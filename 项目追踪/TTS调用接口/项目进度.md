## PRIMARY_CALL_GRAPH
0. Domain + ports define contracts (`src/modules/tts/domain/*`, `src/modules/tts/ports/*`).
1. POST `/api/tts/synthesize` -> `ttsRoutes`
2. `TtsSynthesisService.synthesize()`
3. `VoiceResolver.normalizeRequest()` + `validateText()` + `resolve()`
4. `ProviderCatalog.resolveCanonicalKey(service)` (alias -> canonical)
5. `credentials.isConfigured(providerKey)`
6. `createProvider(adapterKey)`
7. `adapter.synthesizeAndSave(text, runtimeOptions)`
8. adapter internal: select key -> provider call -> report success/failure
9. DTO normalize + HTTP status mapping

## VOICE_DATA_MODEL_NOTE
- VoiceCatalog != VoiceRegistry:
  - VoiceCatalog (`catalog/VoiceCatalog.js`): static runtime catalog backed by `voices/dist/*.json`.
  - VoiceRegistry (`core/VoiceRegistry.js`): dynamic registry path for runtime registration/listing lifecycle.
- TtsQueryService may aggregate query-facing outputs that do not fully equal synthesis runtime selection pipeline.
- AI agents must not assume management listing and synthesis runtime are always identical unless source linkage is verified.

## CREDENTIAL_POOL_FLOW
- init:
  - try YAML loader first (`credentials/sources/providers/*.yaml`)
  - fallback to `.env` default account
- runtime:
  - `selectCredentials(provider, service, context)`
  - execute provider call
  - `reportSuccess` / `reportFailure`
  - health transitions + circuit behavior

## HIGH_PRIORITY_RISKS
- R1_key_mapping_mismatch:
  - symptom: config check pass but adapter creation/use fails
  - trigger: providerKey/adapterKey alias drift
- R2_retry_non_idempotent:
  - symptom: duplicate synthesis/duplicate billing on timeout retry
- R3_metadata_boundary:
  - symptom: internal fields leaked via metadata passthrough
