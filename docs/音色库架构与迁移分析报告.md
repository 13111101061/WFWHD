### 核心设计模式

1. **工厂模式** - VoiceModelRegistry 统一管理所有音色
2. **注册表模式** - 集中式音色信息存储
3. **映射器模式** - ParameterMapper 处理参数转换
4. **策略模式** - 各服务商服务类继承 BaseTtsService
5. **单例模式** - 所有核心组件都采用单例

## 🔧 核心组件详解

### 1. VoiceModelRegistry (音色工厂/注册表)

**位置**: `src/modules/tts/config/VoiceModelRegistry.js`

**职责**:
- 统一管理所有服务商的音色模型
- 提供多种查询方式（按ID、提供商、标签）
- 构建高性能索引

**核心数据结构**:

```javascript
class VoiceModelRegistry {
  constructor() {
    this.models = new Map();      // 模型ID -> 模型详细信息
    this.providers = new Map();    // 提供商 -> Set(模型ID)
    this.tags = new Map();         // 标签 -> {name, models: Set}
    this.isLoaded = false;         // 初始化状态
  }
}
```

**关键方法**:

```javascript
// 核心查询方法
getAllModels()              // 获取所有模型
getModel(id)               // 根据ID获取模型
getModelsByProvider(provider)  // 根据提供商获取模型
getModelsByTag(tag)        // 根据标签获取模型
searchModels(query)        // 搜索模型

// 管理方法
initialize()               // 初始化（加载voiceModels.json）
reload()                   // 重新加载配置
getStats()                 // 获取统计信息
```

**数据来源**:
- 文件: `voiceModels.json`
- 包含所有服务商的音色定义

### 3. VoiceModelMapper (音色映射器)

**位置**: `src/modules/tts/config/VoiceModelMapper.js`

**职责**:
- 系统ID ↔ 提供商参数 双向转换
- 提供验证和过滤功能

**核心方法**:


### 4. ParameterMapper (参数映射器)

**位置**: `src/modules/tts/config/ParameterMapper.js`

**职责**:
- 统一参数 → 各服务商特定API参数 转换
- 参数验证和默认值处理
- 参数标准化（如速度、音量等）

**核心方法**:

```javascript
// 参数映射
mapAndValidate(provider, serviceType, userOptions)
  -> 返回服务商特定的API参数

// 查询方法
getSupportedParameters(provider, serviceType)
getParameterConfig(provider, serviceType, parameterName)
```

**配置来源**:
- 文件: `ProviderConfig.json`
- 定义每个服务商的参数映射规则

## 🔄 数据流转与连接逻辑

### 完整的请求流程

```
1. 用户请求
   POST /api/tts/synthesize
   {
     "service": "aliyun_cosyvoice",
     "text": "你好",
     "voice": "aliyun-cosyvoice-longxiaochun"  // 系统ID
   }

2. 路由层 (ttsRoutes.js)
   ├─ 身份验证
   └─ 转发到 UnifiedTtsController

3. 控制器 (UnifiedTtsController)
   ├─ 解析 service 参数
   ├─ 从 TtsServiceManager 获取对应服务
   └─ 调用 service.synthesize()

4. 服务层 (cosyVoiceService)
   ├─ validateOptions(options)  // ← ParameterMapper
   │   └─ 转换统一参数为服务商参数
   ├─ 调用服务商API
   └─ 返回音频文件

5. 参数映射流程 (ParameterMapper)
   输入: { voice: "系统ID", speed: 1.0 }
   ├─ VoiceModelMapper.systemIdToProviderParams("系统ID")
   │   → 获取 { provider, serviceType, voice, model }
   ├─ ParameterMapper.mapAndValidate()
   │   → 转换为服务商API参数
   └─ 输出: { voice: "longxiaochun_v2", format: "mp3", ... }
```

### 系统ID → 提供商参数转换示例

**场景**: 用户使用系统ID请求合成

```javascript
// 1. 用户输入
{
  service: "aliyun_cosyvoice",
  voice: "aliyun-cosyvoice-longxiaochun",  // 系统ID
  text: "你好世界",
  speed: 1.2,
  volume: 60
}

// 2. VoiceModelMapper 转换
voiceModelMapper.systemIdToProviderParams("aliyun-cosyvoice-longxiaochun")
// 输出:
{
  provider: "aliyun",
  serviceType: "cosyvoice",
  voice: "longxiaochun_v2",      // 提供商音色ID
  model: "cosyvoice-v2",         // 模型版本
  systemId: "aliyun-cosyvoice-longxiaochun"
}

// 3. ParameterMapper 转换参数
parameterMapper.mapAndValidate("aliyun", "cosyvoice", {
  voice: "longxiaochun_v2",
  speed: 1.2,
  volume: 60
})
// 输出:
{
  voice: "longxiaochun_v2",
  format: "mp3",
  sample_rate: 22050,
  volume: 60,          // 根据服务商要求转换
  rate: 1.2,           // 速度参数
  pitch: 1
}

// 4. 最终发送到API
{
  header: { ... },
  payload: {
    model: "cosyvoice-v2",
    parameters: {
      voice: "longxiaochun_v2",
      format: "mp3",
      sample_rate: 22050,
      volume: 60,
      rate: 1.2
    },
    input: { text: "你好世界" }
  }
}
```

```javascript
// UnifiedTtsController.getVoices()
async getVoices(req, res) {
  const { service, provider, gender, tags, search } = req.query;

  if (service) {
    // 传统方式：通过服务类获取
    const ttsService = this.ttsServiceManager.getService(service);
    const voices = await ttsService.getAvailableVoices();
    return res.json({ success: true, data: voices });
  }

  // 新方式：使用音色库
  if (provider) {
    const models = voiceModelRegistry.getModelsByProvider(provider);
    // 应用过滤条件
    if (gender) models = models.filter(m => m.gender === gender);
    if (tags) models = models.filter(m => m.tags.includes(tags));
    return res.json({ success: true, data: models });
  }

  if (search) {
    const models = voiceModelRegistry.searchModels(search);
    return res.json({ success: true, data: models });
  }

  // 默认返回所有
  const allModels = voiceModelRegistry.getAllModels();
  res.json({ success: true, data: allModels });
}
```

#### 阶段5: 前端集成 🎨 可选

**音色选择器组件**:

```javascript
// 音色选择器 - 支持筛选
class VoiceSelector {
  constructor() {
    this.voices = [];
    this.filters = { provider: '', gender: '', tags: '' };
  }

  // 加载音色列表
  async loadVoices() {
    const response = await fetch('/api/tts/voices/models');
    this.voices = await response.json();
  }

  // 筛选
  filter() {
    return this.voices.filter(voice => {
      if (this.filters.provider && voice.provider !== this.filters.provider) return false;
      if (this.filters.gender && voice.gender !== this.filters.gender) return false;
      if (this.filters.tags && !voice.tags.includes(this.filters.tags)) return false;
      return true;
    });
  }

  // 搜索
  search(keyword) {
    return this.voices.filter(voice =>
      voice.name.includes(keyword) ||
      voice.description.includes(keyword)
    );
  }

  // 渲染
  render() {
    const filtered = this.filter();
    return filtered.map(voice => `
      <option value="${voice.id}">
        ${voice.provider} - ${voice.name}
      </option>
    `).join('');
  }
}
```

## 🎓 总结

### 核心理念
**"配置优于代码，统一优于分散"**

将分散在各服务类中的音色数据集中到 `voiceModels.json`，通过 `VoiceModelRegistry` 统一管理，是本次重构的核心。

1. **可维护性提升** - 修改音色信息不再需要改代码

3. **架构清晰** - 职责分离，各组件专注自己的功能
