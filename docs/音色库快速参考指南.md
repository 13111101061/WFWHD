# 音色库快速参考指南

## 🎯 一分钟了解音色库架构

### 核心概念

```
音色库 = 一个大仓库 (voiceModels.json)
         ↓
    管理员 (VoiceModelRegistry)
         ↓
    导购员 (VoiceModelMapper)
         ↓
    翻译官 (ParameterMapper)
         ↓
   各个柜台 (服务商服务类)
```

### 实际例子

```javascript
// 用户想用腾讯云的"亲亲"音色

// 1. 用户输入 (系统ID - 全局唯一)
"tencent-tts-101001"

// 2. VoiceModelRegistry 查找
{
  id: "tencent-tts-101001",
  name: "亲亲",
  provider: "tencent",
  voiceId: "101001",         // 腾讯云的ID
  model: "1",
  gender: "female",
  tags: ["popular", "sweet"]
}

// 3. VoiceModelMapper 转换
provider: "tencent"
serviceType: "tts"
voice: "101001"              // ← 腾讯云需要的ID
model: "1"

// 4. ParameterMapper 处理参数
{
  VoiceType: 101001,         // ← 腾讯云API字段名
  Codec: "mp3",
  SampleRate: 16000,
  Speed: 1,
  Volume: 0
}

// 5. TencentTtsService 调用API
发送到腾讯云服务器...
```

## 📦 三大核心组件

### 1. VoiceModelRegistry (仓库管理员)

**位置**: `src/modules/tts/config/VoiceModelRegistry.js`

**作用**: 管理所有音色的"大仓库"

**常用方法**:
```javascript
// 获取所有音色
voiceModelRegistry.getAllModels()

// 按服务商获取
voiceModelRegistry.getModelsByProvider('tencent')
// → 返回腾讯云的21个音色

// 按标签获取
voiceModelRegistry.getModelsByTag('sweet')
// → 返回所有"甜美"标签的音色

// 搜索
voiceModelRegistry.searchModels('龙')
// → 返回名字包含"龙"的音色

// 获取统计
voiceModelRegistry.getStats()
// → { totalModels: 52, totalProviders: 6, ... }
```

### 2. VoiceModelMapper (导购员)

**位置**: `src/modules/tts/config/VoiceModelMapper.js`

**作用**: 把系统的"商品ID"转换成各服务商的"条形码"

**常用方法**:
```javascript
// 系统ID → 提供商参数
voiceModelMapper.systemIdToProviderParams('tencent-tts-101001')
// → {
//     provider: 'tencent',
//     serviceType: 'tts',
//     voice: '101001',           // 腾讯云的ID
//     model: '1'
//   }

// 提供商ID → 系统ID
voiceModelMapper.providerIdToSystemId('101001', 'tencent')
// → 'tencent-tts-101001'

// 验证系统ID
voiceModelMapper.validateSystemId('tencent-tts-101001')
// → true
```

### 3. ParameterMapper (翻译官)

**位置**: `src/modules/tts/config/ParameterMapper.js`

**作用**: 把统一的参数翻译成各服务商能听懂的语言

**常用方法**:
```javascript
// 参数映射
parameterMapper.mapAndValidate('tencent', 'tts', {
  voice: '101001',
  speed: 1.5,
  volume: 50
})
// → {
//     VoiceType: 101001,      // 腾讯云字段名
//     Speed: 1.5,
//     Volume: 50,
//     Codec: 'mp3',           // 自动添加默认值
//     SampleRate: 16000
//   }

// 获取支持的参数
parameterMapper.getSupportedParameters('tencent', 'tts')
// → 返回腾讯云支持的所有参数列表
```

## 📝 voiceModels.json 结构

### 完整示例

```json
{
  "models": [
    {
      "id": "tencent-tts-101001",           // 系统ID (唯一)
      "name": "亲亲",                         // 显示名称
      "provider": "tencent",                 // 服务商
      "service": "tts",                      // 服务类型
      "voiceId": "101001",                   // 提供商音色ID
      "model": "1",                          // 模型版本
      "gender": "female",                    // 性别
      "languages": ["zh-CN"],                // 语言
      "tags": ["popular", "sweet"],          // 标签
      "description": "温柔甜美女声"           // 描述
    }
  ],
  "tags": {
    "popular": {
      "name": "热门",
      "color": "#ff6b6b"
    },
    "sweet": {
      "name": "甜美",
      "color": "#ff6bb6"
    }
  }
}
```

### 字段说明

| 字段 | 说明 | 示例 | 必填 |
|------|------|------|------|
| `id` | 系统ID，全局唯一 | `"tencent-tts-101001"` | ✅ |
| `name` | 显示名称 | `"亲亲"` | ✅ |
| `provider` | 服务商 | `"tencent"` | ✅ |
| `service` | 服务类型 | `"tts"` | ✅ |
| `voiceId` | 提供商的音色ID | `"101001"` | ✅ |
| `model` | 模型版本 | `"1"` | ❌ |
| `gender` | 性别 | `"female"` | ❌ |
| `languages` | 支持的语言数组 | `["zh-CN"]` | ❌ |
| `tags` | 标签数组 | `["sweet", "popular"]` | ❌ |
| `description` | 描述 | `"温柔甜美女声"` | ❌ |

## 🔄 迁移服务类的步骤

### 模板代码

```javascript
// 1. 添加导入
const { voiceModelRegistry } = require('../config/VoiceModelRegistry');

// 2. 修改 getAvailableVoices()
async getAvailableVoices() {
  try {
    // 初始化注册中心
    if (!voiceModelRegistry.isLoaded) {
      await voiceModelRegistry.initialize();
    }

    // 获取当前服务商的模型
    const models = voiceModelRegistry.getModelsByProvider('服务商名');
    const filteredModels = models.filter(m => m.service === '服务类型');

    // 转换为API格式
    return filteredModels.map(model => ({
      id: model.voiceId,              // 提供商ID (向后兼容)
      systemId: model.id,             // 系统ID (新功能)
      name: model.name,
      gender: model.gender,
      language: model.languages[0],
      model: model.model,
      _modelInfo: model               // 完整信息
    }));

  } catch (error) {
    console.error('从音色库获取失败:', error);

    // 降级: 返回硬编码数据
    return [
      { id: 'xxx', name: 'xxx', ... }
    ];
  }
}
```

## ⚠️ 常见问题

### Q1: 系统ID和提供商ID的区别?

```
提供商ID: 各服务商自己的ID,可能重复
- 腾讯云: "101001"
- 阿里云: "101001"  (可能相同)

系统ID: 全局唯一,包含服务商信息
- "tencent-tts-101001"  (腾讯云)
- "aliyun-cosyvoice-longxiaochun"  (阿里云)
```

### Q2: 为什么要用系统ID?

1. **避免冲突**: 不同服务商可能有相同的音色ID
2. **统一管理**: 前端不需要关心是哪个服务商
3. **易于扩展**: 新增服务商不会影响现有代码

### Q3: 如何判断ID类型?

```javascript
function isSystemId(id) {
  // 系统ID通常包含"-"分隔符
  return id && id.includes('-');
}

// 使用
if (isSystemId(options.voice)) {
  // 系统ID,需要转换
  const params = voiceModelMapper.systemIdToProviderParams(options.voice);
  apiParams.voice = params.voice;
}
```

### Q4: 降级处理是什么?

当音色库加载失败时,使用硬编码的备份数据,确保服务不中断。

```javascript
try {
  // 优先使用音色库
  return await getVoicesFromRegistry();
} catch (error) {
  // 降级到硬编码数据
  console.warn('使用备份数据');
  return getHardcodedVoices();
}
```
