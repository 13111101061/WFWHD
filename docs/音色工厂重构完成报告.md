## ⚠️ 注意事项

### 缓存删除的影响

**之前：**
- 相同文本 + 相同参数 → 返回缓存
- 响应快，但占用磁盘空间
- 缓存一致性问题

**现在：**
- 每次都调用API
- 响应略慢（取决于服务商）
- 无缓存问题，结果总是最新的

**建议：**
- 如需缓存，可在应用层实现（Redis等）
- 或在服务商层面实现（如果支持）

### 向后兼容性

**新旧字段名同时支持：**
```json
{
  "id": "cosyvoice-longxiaochun",      // 旧字段（保留）
  "systemId": "cosyvoice-longxiaochun", // 新字段（推荐）
  "voiceId": "longxiaochun_v2",         // 旧字段（保留）
  "providerVoiceId": "longxiaochun_v2"  // 新字段（推荐）
}
```

**迁移策略：**
1. 新代码使用新字段名
2. 旧代码继续工作（兼容）
3. 逐步迁移，不强制切换
